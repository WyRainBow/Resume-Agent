# 简历优化功能 - 使用指南

## 功能概述

简历优化功能通过 AI 诊断和对话式引导，帮助用户完善简历内容。

---

## 快速开始

### 1. 访问优化页面

**方式一：从简历列表**
1. 访问简历列表页面：`http://localhost:5173/dashboard`
2. 找到任意简历卡片
3. 点击 **✨ 优化按钮**（紫色图标）

**方式二：直接访问**
```
http://localhost:5173/resume/optimize/{resume_id}
```

---

### 2. 自动诊断流程

页面加载后，系统会自动：

1. ✅ **读取简历数据** - 从 localStorage 加载
2. ✅ **执行多维度诊断** - 检查完整性、内容质量、结构、匹配度
3. ✅ **生成诊断报告** - 显示评分和主要问题
4. ✅ **提供优化选项** - 智能引导用户从哪里开始

---

## 功能详解

### 📊 诊断报告卡片

**显示内容**：
- **总体评分**：0-100 分，带颜色编码
  - 🎉 80+ 分：绿色（优秀）
  - 👍 60-80 分：黄色（良好）
  - 💪 40-60 分：橙色（需改进）
  - 📝 <40 分：红色（需大幅改进）

- **主要问题列表**：按严重程度排序
  - ❌ Critical（关键）：必须解决
  - ⚠️ High（重要）：建议尽快解决
  - 💡 Medium（一般）：可以后续优化

- **优化路径**：系统推荐的优化步骤

### 🎯 引导选项卡片

**优化选项**：
1. **个人总结** - 先写好简历的"开场白"
2. **工作经历** - 填充您的实战经验
3. **项目经历** - 展示项目成果
4. **教育经历** - 补充教育背景
5. **技能** - 列出专业技能
6. **自动优化** - 按照专业建议系统性优化

每个选项都有：
- 📌 优先级标记
- 💡 推荐理由
- 🔗 相关模块

## 对话示例

### 示例 1：优化个人总结

```
👤 用户：点击"个人总结"选项

🤖 AI：好的，个人总结是HR看简历的第一眼！让我帮您优化。

请告诉我：
1. 您的目标岗位是什么？（如：前端工程师、产品经理）
2. 您的核心优势是什么？（如：3年前端开发经验、熟悉React）

👤 用户：我想应聘前端工程师，有3年React开发经验

🤖 AI：很好！让我帮您生成一个优化后的个人总结：

"资深前端工程师，3年React开发经验，擅长使用现代前端技术栈构建高性能Web应用。熟悉React、Vue等框架，具备良好的代码规范和团队协作能力。热衷于学习新技术，追求代码质量和用户体验。"

您觉得怎么样？需要调整吗？
```

### 示例 2：优化工作经历

```
👤 用户：帮我优化工作经历

🤖 AI：好的！在优化工作经历时，我建议您：

💡 **STAR法则**：
- **S**ituation：当时什么背景？
- **T**ask：负责什么任务？
- **A**ction：具体做了什么？
- **R**esult：取得了什么成果？

请告诉我您最近一份工作的：
1. 公司名称和职位
2. 主要职责
3. 使用的技术
4. 取得的成果（最好有数据）

👤 用户：我在ABC公司做前端开发，负责公司官网重构，使用React和TypeScript

🤖 AI：很好的开始！为了更好地展示您的价值，建议补充：

💡 **量化数据**：
- 网站性能提升了多少？
- 开发周期多长？
- 团队规模多大？

👤 用户：性能提升了40%，3个月完成，5人团队

🤖 AI：太棒了！我帮您重新组织：

"在ABC公司担任前端开发工程师，负责公司官网的重构工作。使用React和TypeScript技术栈，带领5人团队在3个月内完成项目。通过优化代码架构和资源加载策略，将网站性能提升40%，显著改善用户体验。"
```

## 技术实现

### 前端架构

```typescript
ResumeOptimizationPage
├── 状态管理
│   ├── messages - 对话消息列表
│   ├── resumeData - 简历数据
│   ├── isProcessing - 处理状态
│   └── currentModule - 当前优化模块
├── 核心功能
│   ├── loadResumeData() - 从localStorage加载
│   ├── handleDiagnose() - 触发诊断
│   ├── handleSendMessage() - 发送消息
│   └── handleChoiceClick() - 点击选项
└── 组件
    ├── DiagnosisReportCard - 诊断报告
    ├── GuidanceChoicesCard - 引导选项
    ├── ResumePreviewPanel - 简历预览
    └── LoadingSpinner - 加载动画
```

### 后端架构

```python
ResumeOptimizationAgent
├── 诊断系统
│   ├── ResumeDiagnosis - 4维度诊断
│   ├── DiagnosisReport - 报告生成
│   ├── GuidanceEngine - 引擎和决策
│   └── FollowUpSystem - STAR追问
├── 意图识别
│   ├── diagnose - 诊断意图
│   ├── optimize_module - 优化模块
│   └── provide_info - 提供信息
└── 响应生成
    ├── 流式响应（SSE）
    ├── 消息类型
    └── 错误处理
```

### API 端点

#### POST /api/resume-optimization/diagnose
**请求**：
```json
{
  "resume_id": "resume_xxx",
  "resume_data": { /* 简历数据 */ }
}
```

**响应**：
```json
{
  "success": true,
  "report": { /* 诊断报告 */ },
  "message": "您的简历需要一些优化...",
  "choices": [ /* 引导选项 */ ]
}
```

#### POST /api/resume-optimization/chat
**请求**：
```json
{
  "resume_id": "resume_xxx",
  "message": "帮我优化个人总结",
  "session_id": "session_xxx",
  "stream": true
}
```

**响应**（流式 SSE）：
```
data: {"type": "text", "content": "好的，让我帮您...", "role": "assistant"}

data: {"type": "followup", "content": "请告诉我您的目标岗位", "role": "assistant"}

data: {"type": "update_success", "path": "...", "message": "✅ 已更新！", "role": "assistant"}
```

## 最佳实践

### 1. 准备工作

✅ **推荐**：
- 完善基本信息（姓名、联系方式）
- 有大致的职业方向
- 准备好关键数据（项目成果、工作年限）

❌ **避免**：
- 空白简历开始优化
- 过于模糊的描述
- 夸大其词

### 2. 优化顺序

**推荐路径**：
1. **个人总结** → 确立定位
2. **工作经历** → 展示核心能力
3. **项目经历** → 证明实战能力
4. **技能** → 补充技术栈
5. **教育** → 背景支撑

### 3. 对话技巧

- ✅ 提供具体信息（有数据更好）
- ✅ 诚实描述经历
- ✅ 主动反馈AI生成的内容
- ❌ 不要只说"帮我优化XXX"
- ❌ 不要提供虚假信息

## 常见问题

### Q: 诊断评分很低怎么办？

A: 不要担心！评分低意味着还有很大优化空间。按照系统建议的路径一步步来：
1. 优先解决"关键"问题
2. 逐步完善各个模块
3. 重新诊断查看进步

### Q: AI生成的内容不太准确？

A: 可以：
1. 继续对话，告诉AI需要调整的地方
2. 提供更多具体信息
3. 或者手动修改后再继续优化其他模块

### Q: 可以跳过某些模块吗？

A: 当然可以！系统设计就是**平衡引导**：
- 随时可以切换到其他模块
- 可以直接告诉AI"我想优化XX模块"
- 提供退出选项

### Q: 优化会保存吗？

A: 当前版本：
- ✅ 可以看到预览效果
- ⏳ 自动保存功能开发中
- 建议：优化完成后手动保存
