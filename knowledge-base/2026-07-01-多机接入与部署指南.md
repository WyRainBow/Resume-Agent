# 另一台电脑接入开发 + 部署指南

> 目标：在新电脑上搭好 Resume-Agent 本地开发环境，并能把改动部署到生产。
> 生成日期：2026-07-01

## 核心结论（先看这条）

**部署 = `git push origin main`。** 服务器有 cron（每 2 分钟）自动拉 main、重新 build、重启 pm2。
**你只要能 push 到 GitHub 的 main，就完成部署，不需要手动登服务器。**

SSH 到服务器只用于**调试或手动兜底部署**，日常不需要。

---

## 一、前置工具（新电脑先装）

| 工具 | 版本 | 说明 |
|---|---|---|
| git | 任意 | 克隆 / 推送代码 |
| Node.js | v20（服务器用 v20.20.2） | 前端 + web 构建，建议用 nvm |
| Python | 3.11 | 后端 |
| uv / pip | - | 后端依赖 |
| pm2 | 仅服务器需要 | 本地不用 |

## 二、GitHub 访问（必需——决定能否 push 部署）

仓库：`git@github.com:WyRainBow/Resume-Agent.git`

新电脑要能 push 到它，二选一：

**方式 A：GitHub SSH Key（推荐）**
1. `ssh-keygen -t ed25519 -C "你的邮箱"`
2. 把 `~/.ssh/id_ed25519.pub` 加到 GitHub 账号（WyRainBow）→ Settings → SSH Keys
3. 测试：`ssh -T git@github.com`

**方式 B：HTTPS + Personal Access Token**
- clone 用 `https://github.com/WyRainBow/Resume-Agent.git`，push 时用 PAT 当密码

## 三、SSH 部署密钥（可选——仅用于登服务器调试 / 手动兜底部署）

服务器 SSH：`ssh -i ~/.ssh/resume_deploy -p 2222 root@106.53.113.137`
（**端口是 2222，不是默认 22**）

从**原电脑**把这两个文件复制到新电脑同路径：
- `~/.ssh/resume_deploy`（私钥）
- `~/.ssh/resume_deploy.pub`（公钥）

复制后设权限：`chmod 600 ~/.ssh/resume_deploy`

> ⚠️ 私钥是敏感文件，用 U 盘 / 加密方式传，别发聊天 / 邮件 / 云盘明文。

## 四、克隆项目 + 环境配置

```bash
git clone git@github.com:WyRainBow/Resume-Agent.git
cd Resume-Agent
```

**环境变量**（不进 git，需从原电脑或服务器复制）：
- 根目录 `.env`：后端用（`DASHSCOPE_API_KEY`、`COS_*`、`DATABASE_URL`、`JWT_SECRET_KEY` 等）
- `web/.env.local`：web 登录用（BetterAuth、Google OAuth、`BETTER_AUTH_DATABASE_URL` 等）
- `frontend/.env.local`：前端用（如 `VITE_API_VIA_AUTH_WEB=true`），如有

> 服务器上的位置：`/www/wwwroot/Resume-Agent/.env`、`web/.env.local`。可 SSH 过去 scp 下来参考（含密钥，注意保密）：
> ```bash
> scp -i ~/.ssh/resume_deploy -P 2222 root@106.53.113.137:/www/wwwroot/Resume-Agent/.env ./
> ```

**装依赖 + 本地启动**：
```bash
# 后端（:9000）
uv pip install -r requirements.txt        # 或 python -m venv .venv && .venv/bin/pip install -r requirements.txt
python -m uvicorn backend.main:app --host 127.0.0.1 --port 9000

# 前端（:5173）
cd frontend && npm install && npm run dev

# web 登录（:3000）
cd web && npm install && npm run dev
```
访问：前端 http://127.0.0.1:5173、后端 http://127.0.0.1:9000/docs、web http://127.0.0.1:3000

## 五、部署（关键）

### 日常部署：push main 即可
```bash
git add -A
git commit -m "你的改动说明"
git push origin main
```
服务器 cron（`*/2 * * * * /www/wwwroot/Resume-Agent/auto-deploy.sh`）会在 2 分钟内自动：
1. `git fetch origin main`；若和本地 HEAD 不一致则 `git reset --hard origin/main`
2. `venv/bin/pip install -r requirements.txt`
3. `frontend` + `web` 各 `npm install && npm run build`
4. `pm2 restart all --update-env`

**web（登录）另由 Vercel** 在 push main 后自动部署。

### 手动兜底部署（cron 失败时，比如撞 GitHub 网络超时 / build 报错）
```bash
ssh -i ~/.ssh/resume_deploy -p 2222 root@106.53.113.137
cd /www/wwwroot/Resume-Agent
bash auto-deploy.sh                    # 直接跑部署脚本
tail -n 50 /var/log/auto-deploy.log    # 看部署日志
```

## 六、验证部署生效

```bash
# 生产是否到你 push 的 commit + pm2 是否 online
ssh -i ~/.ssh/resume_deploy -p 2222 root@106.53.113.137 \
  'cd /www/wwwroot/Resume-Agent && git log --oneline -1 && pm2 list'

# 生产可访问
curl -o /dev/null -w "%{http_code}\n" https://resumegenkk.xyz
```

## 常见问题

- **push 后生产没更新？** 等满 2 分钟（cron 周期）；仍没有就 SSH 看 `/var/log/auto-deploy.log`，多半是 GitHub 网络超时或 build 报错，手动跑一次 `bash auto-deploy.sh`。
- **SSH 连不上？** 确认用了 `-p 2222` 和 `-i ~/.ssh/resume_deploy`，且私钥权限是 600。
- **本地起不来？** 先确认 `.env` / `web/.env.local` 已从原电脑或服务器拷过来。
