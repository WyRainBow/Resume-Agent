# 07 上线与打磨清单

来源页码：Page 040-060, Page 090-100

## 最小上线闭环

一个能收钱的海外 AI MicroSaaS，至少要跑通：

- Landing Page
- Google 登录
- 免费用户体验
- 核心 AI 功能
- 付费按钮
- Checkout
- Webhook
- 数据库订阅状态
- 付费权限生效
- 成功页/失败页
- 部署到 Vercel 或其他生产环境

## 什么叫真正上线

部署成功不等于上线。

真正上线意味着一个真实用户可以：

1. 通过页面理解产品。
2. 登录。
3. 使用免费功能。
4. 碰到清楚的付费引导。
5. 完成支付。
6. 支付后自动获得权益。
7. 再次访问时权益仍然存在。

如果中间任何一步断了，都只是“代码部署了”，还不是商业化上线。

## 支付上线前检查

- 使用测试模式完成全流程。
- 测试卡支付成功。
- Webhook 地址是公网地址。
- Webhook 签名验证通过。
- Webhook 路由能处理重复通知。
- 用户订阅状态能写进数据库。
- 免费和付费权限能立刻变化。
- Vercel 环境变量配置完整。
- 正式环境 API URL 和测试环境 API URL 没有混用。

## 认证上线前检查

- Google OAuth consent screen 配置完整。
- 本地和正式域名的 redirect URI 都已添加。
- `BETTER_AUTH_URL` 在本地和线上分别正确。
- 登录后刷新页面 session 不丢。
- 退出登录后付费资源不可访问。
- Vercel 环境变量包含认证相关 secret。
- 生产环境 secret 足够长，不使用示例值。

## 产品体验检查

- 首屏一句话是否讲清价值。
- CTA 是否清楚。
- 免费用户是否知道为什么要升级。
- 付费用户是否看到权益变化。
- 错误状态是否有友好提示。
- 移动端是否可用。
- 页面加载速度是否能接受。

## AI 功能检查

- API Key 配置在服务端。
- 前端不会泄露模型 Key。
- 上传图片大小有限制。
- 生成中有 loading 状态。
- 失败后有明确提示。
- 生成结果保存成功。
- 免费用户水印或限制生效。
- 付费用户无水印或高权限生效。
- API 失败不会扣错额度。

## 代码与工程检查

- 本地能启动。
- 类型检查通过。
- 构建通过。
- 核心 API 错误处理合理。
- 敏感密钥不提交到 GitHub。
- `.env.local` 和 Vercel 环境变量一致。
- 数据库迁移已执行。
- 日志能帮助排查支付和 AI 生成失败。

## 数据与后台检查

- 用户表能看到新注册用户。
- 订阅表能看到支付成功后的记录。
- 支付事件有日志或 raw event。
- 生成历史能追踪到用户。
- 管理员能手动查看或修正用户状态。
- 退款、取消订阅至少有处理预案。

## 后续打磨方向

课程提到的打磨内容：

1. Landing Page 控件和样式。
2. 多种登录方式，例如邮箱登录。
3. 不同等级付费用户的权限。
4. 积分购买功能，不止包月。
5. Logo 和 icon。
6. 文案精细化。
7. SEO。
8. 页面性能。
9. 博客文章。
10. 品牌故事。

## 打磨优先级

先打磨影响商业闭环的东西：

1. 首屏文案。
2. 登录路径。
3. 免费额度和付费引导。
4. 支付成功后的权益变化。
5. 核心功能稳定性。

再打磨增长相关：

1. SEO。
2. 博客。
3. 案例。
4. 社交分享图。
5. 品牌故事。

最后再打磨高级功能：

1. 多套餐。
2. 团队账户。
3. 分销。
4. 多租户。
5. 更复杂后台。

## 常见风险

伪需求：
功能做出来，但没有真实用户愿意付费。

支付没闭环：
能打开收银台，但支付后会员状态没有更新。

Webhook 不稳定：
本地能测，线上收不到；或者收到了但验签失败。

权限没隔离：
免费用户绕过限制，或付费用户无法使用权益。

过早追求复杂架构：
还没有验证需求，就花大量时间做后台、分销、多租户。

## 我的提炼

上线不是“部署成功”，而是“真实用户能从看到页面、登录、试用、付费，到拿到权益”。如果这条链路没有跑通，产品还没有真正上线。
