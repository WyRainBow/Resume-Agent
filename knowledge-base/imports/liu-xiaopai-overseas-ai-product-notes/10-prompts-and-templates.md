# 10 提示词模板

## 产品想法验证提示词

```text
我有一个 AI MicroSaaS 想法：

目标人群：
具体场景：
想解决的问题：
现有替代方案：
我初步想做的功能：

请你按 MicroSaaS 的标准帮我判断：
1. 这个人群是否足够具体？
2. 这个问题是否高频、具体、痛？
3. 用户为什么愿意付钱？
4. 它更适合订阅、积分，还是一次性付费？
5. 最大的伪需求风险是什么？
6. 我应该如何做最小验证？
```

## 支付接入提示词

```text
我正在给 Next.js MicroSaaS 接入订阅支付。

支付平台：
测试模式是否已开启：
产品 ID：
API Key 是否已配置：
Webhook Secret 是否已配置：
期望的 Webhook URL：
数据库：
用户表/订阅表现状：

请你先阅读项目结构和官方文档，再给我一个实现计划。
不要马上写代码。

计划里必须包含：
1. 需要新增或修改的路由。
2. 需要新增或修改的数据库字段。
3. Checkout 创建流程。
4. Webhook 验签和状态更新流程。
5. 前端权限变化。
6. 本地和线上验收步骤。
```
