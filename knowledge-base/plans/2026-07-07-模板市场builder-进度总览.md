## 一、这是什么

抄 Resume-Matcher(RM),在我方站内做一套 **Swiss 风格的简历 Builder 模块**,独立于现有工作台,消费我方简历数据。三个页面 + 一套模板 + 一套模型配置后台。

| 路由 | 页面 | 权限 |
|---|---|---|
| `/builder/dashboard` | 简历卡片墙(选简历进 Builder) | 登录可见,设置入口仅管理员 |
| `/builder` · `/builder/:id` | 简历编辑器(排版 + 内容 + 预览 + 下载) | 公开 |
| `/builder/settings` | 系统设置 / 模型配置 | 仅管理员 |

## 二、已完成 ✅

### 1. 模板系统(100% 复刻)
- **5 套模板**:单栏 / Modern / LaTeX / Clean / Vivid(双栏两套按用户决策不做);tsx 逐行移植、css 与 RM 源逐字节对齐
- **Vivid 中文名两色调**:纯中文名按「姓 / 名」双色(含 16 个复姓),补齐 RM 只认空格的短板
- **排版控制面板**:模板缩略图、强调色、A4/Letter、边距滑块、间距 / 字号 5 档、Serif/Sans/Mono、紧凑模式、生效值汇总、恢复默认
- **分页实时预览**:真实 mm 纸张、自动分页避断、边距参考线、缩放、页码指示

### 2. 内容编辑(P1-1,把 Builder 变真编辑器)
- **9 个模块表单**:基本信息 / 教育 / 经历 / 项目 / 开源 / 技能 / 荣誉 / 自我评价 / 自定义模块
- **模块管理**:折叠 / 显隐 / 排序(↑↓ + 拖拽双通道)/ 新增自定义模块 / **自定义模块重命名**(2026-07-07 新增,内置模块仍只读);富文本直接复用工作台 RichEditor(TipTap)
- **架构**:真相源 = v2 ResumeData,表单直接编辑 v2 字段、渲染经 adapter 派生 —— 无有损反向转换,保存不丢任何字段
- **保存链路**:SAVE 写 `saveResume`(排版设置随简历入库 `globalSettings.builderSettings`);RESET 回滚上次保存;未保存徽标 + 关页拦截;AI 优化入口(跳 Coco `/agent/:id`)

### 3. Dashboard(复刻 RM /dashboard)
- 格纸画布 + 5 列黑缝卡片墙;每卡:monogram 色块(中文名适配)、标题、「状态: 就绪」、更新日期、置顶标记
- 「创建简历」新建卡、filler 补格、底栏(产品 logo + 简历计数 + 我的简历入口)
- **Bugfix(2026-07-08)**:「设置」按钮点击没反应——`DashboardPage.tsx` 只按本地存储角色决定按钮是否显示,未校验登录态是否仍然有效;而 `App.tsx` 放行 `/builder/settings` 路由额外要求 `isAuthenticated`,登录态失效(session 过期等)时两处判断不一致,按钮点击后被路由静默重定向回 dashboard,像是"没反应"。改为点击时用 `useAuth()` 的实时 `isAuthenticated` 判断:未登录 → `openModal('login')` 弹出登录框(复用全局登录弹窗,非新范式);已登录才跳 `/builder/settings`

### 4. Settings(复刻 RM /settings 中文版完整形态)

## 四、决策留痕(避免接手人重复纠结)

| 决策 | 结论 | 理由 |
|---|---|---|
| 双栏两套模板 | 不做 | 用户明确不要 |
| vivid 中文名双色 | 做了(姓/名切分) | 招牌特性,主力用户是中文名 |
| 模板设置存哪 | 随简历入库 + localStorage 缓存 | 换设备不丢 |
| Settings 提供商 | 仅 DeepSeek | 用户要求只留国产主力 |
| 重置数据库按钮 | UI 抄 + 禁用 | 多用户生产库,一键清空 = 删所有人简历 |
| 界面语言 | 中文(大标题 / 模板名留英文) | 用户拍板;英文大写是设计语言核心 |
| 反向适配器 | 不做 | 真相源用 v2 数据,表单直接编辑,免有损转换 |
