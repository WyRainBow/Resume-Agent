## 三、数据模型适配(核心设计)

RM 原形状 `{personalInfo, summary, workExperience, education, personalProjects, additional, sectionMeta, customSections}` 按我方简历扩展为:

```ts
BuilderResumeData {
  personalInfo   // ← basic(blog 含 github.com → github,否则 website)
  summary: string[]        // ← selfEvaluation HTML→bullets;1 条=正文段,多条=列表
  workExperience // ← experience[](company/position/date/details→bullets)
  education      // ← education[](degree='学位 · 专业 · GPA';description→bullets)
  personalProjects // ← projects[](link→website)
  openSource: Project[]    // 新增 section,复用各模板 projects 渲染(repo→github pill)
  skills: string[]         // ← skillContent HTML→bullets(RM 的 additional 弃用,句子型技能不适合 pill/逗号连接)
  awards: string[]         // ← awards[] → '<strong>标题</strong> · 颁发方 · 日期:描述'
  customSections           // ← customData(竞赛科研等,itemList)
  sectionMeta    // ← menuSections(title→displayName 中文,enabled→isVisible,order→order)
}
```

- 富文本 HTML→bullets:DOMParser 取 `<li>` innerHTML;无列表则按 `<p>` 段;保留行内 strong/em/u/a 由 SafeHtml 渲染
- 各模板 renderSection switch 增加 `openSource`(=projects 视觉)/`skills`/`awards`(=renderBullets)三个 case,删除 RM 的 `additional` 死分支
- 边界校验:storage 读取处对 `saved.data` 做字段级默认合并(与 useResumeData 同模式)
