# Coco 对话内 QQ 邮箱发送简历 实施计划

> 设计文档:`knowledge-base/specs/2026-07-09-coco-qq邮箱发送-design.md`
> 分支:`feature/coco-email-send`(从 main 切出)

**目标**:管理员在 Coco 对话里说"把简历发到 xxx@qq.com",经二次确认后,后端用管理员自己配置的 QQ 邮箱 SMTP 发送当前简历 PDF。

**架构**:凭证 Fernet 加密存新表 `email_credentials`;新路由 `GET/PUT/DELETE /api/email/credential`(仅 admin);新 Agent 工具 `send_resume_email`(仅 admin 会话注册,`confirm` 两段式);前端 Composer 工具栏新增 Mail 图标 + 连接面板(仅 admin 可见)。

## 全局约束

- 不引入新 pip 依赖(`cryptography` 已由 `python-jose[cryptography]` 带入)
- 加密密钥环境变量:`EMAIL_CREDENTIAL_ENC_KEY`,缺失时相关功能报错而非明文降级
- SMTP 固定 `smtp.qq.com:465` SSL
- 频率限制:同用户每小时 5 次发送尝试(内存计数)
- 所有服务端入口双重校验 `role == 'admin'`

## Task 5: Agent 工具 send_resume_email + 管理员条件注册

- Create: `backend/agent/tool/send_resume_email_tool.py`
- Modify: `backend/agent/tool/__init__.py`(导出)、`backend/agent/agent/manus.py`(新增 `is_admin: bool = False`、`user_id: Optional[int] = None` 字段;`_build_tool_collection` 仅 admin 追加该工具;`_inject_tool_context` 注入 `user_id`)、`backend/agent/web/routes/stream.py:136`(`Manus(session_id=..., is_admin=_is_admin(user), user_id=user.id)`)
- 工具逻辑:参数 `to_email`(必填)/`subject`/`message`/`confirm`(默认 false)
  - `confirm=false` → 不发送,返回确认文案;工具 description 写明"必须先以 confirm=false 调用,待用户明确回复确认后才允许 confirm=true"
  - `confirm=true` → 依次:DB 查 user role 二次校验 → 频率限制 → 邮箱格式校验 → 查凭证(无则引导去连接)→ `ResumeDataStore.get_data(session_id)` 取简历(无则提示先加载)→ 复用 `backend/routes/pdf.py` 的 `_prepare_latex_content` + `_compile_pdf_bytes` 生成 PDF → 解密凭证 → `email_sender` 发送 → 成功/失败文案
- Test: `backend/tests/test_send_resume_email_tool.py`(mock SMTP + PDF:无凭证、confirm 两段式、超限、非 admin Manus 工具列表不含该工具)
