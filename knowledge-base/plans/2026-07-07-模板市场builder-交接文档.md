## 二、快速上手


- `http://127.0.0.1:5173/builder` —— 载入「当前简历」(resumeStorage 的 currentResumeId)

## 四、必须遵守的约定(改动前先读)

1. **模板 css module 与 RM 源逐字节对齐**:改样式前先对照 `reference/projects/Resume-Matcher/apps/frontend/components/resume/styles/` 的同名文件,除非明确要偏离 RM 视觉,否则不改
2. **数据模型契约**(`types.ts` BuilderResumeData,区别于 RM 原版的点):
   - `summary: string[]`(自我评价;1 条渲染正文段、多条渲染 bullets)
   - `education[].description: string[]`(bullets,RM 原是单段字符串)
   - 新增一等 section:`openSource: Project[]`(渲染复用 projects,repo→github pill)、`skills: string[]`、`awards: string[]`(均 bullets;**不要**用 RM 的 additional 行内分类/skill-pill——我方技能是句子型)
   - `sectionMeta` 由我方 `menuSections` 映射(displayName 中文、enabled→isVisible、order 驱动顺序)
   - 每套模板的 renderSection switch 都有 `openSource/skills/awards` 三个 case;新增模板必须同样覆盖
3. **bullets 内容是行内 HTML**,渲染必须经 `<SafeHtml html={...} />`,不许裸 dangerouslySetInnerHTML
4. **Swiss token 换算表**(RM Tailwind v4 token → 我方 arbitrary class,新组件沿用):
   `bg-background→bg-[#F0F0E8]`、`bg-secondary→bg-[#E5E5E0]`、`paper-tint→#F1F2F5`、`steel-grey→#878E99`、`ink-soft→#444850`、`shadow-sw-xs/sm/default/card/lg → 1/2/4/6/8px 硬黑阴影`、`rounded-none` 一律保留
5. **不做双栏模板**(swiss-two-column / modern-two-column 用户明确不要);vivid 内部是 63/37 布局但属于保留的 5 套之一
6. 模板设置持久化 key:`builder_settings:{resumeId}`(localStorage)
7. 遵守仓库 `CLAUDE.md` 全部纪律(分支上开发、完成即 commit、UI 必须浏览器实测、不顺手重构)

## 六、坑与注意事项

1. `npm run build` 只是 vite 构建(不含 tsc);类型检查用 `npx tsc --noEmit`,且仓库存在**大量历史类型错误**——只保证 `pages/Builder` 目录为零
2. Tailwind 是 **v3**:RM(v4)的 `h-18` 这类类不存在,已换 `h-[4.5rem]`;新抄组件遇到同类问题照此处理
3. usePagination 的选择器用 `[class*="resume-item"]` 匹配(CSS Modules 类名带 hash),模板里凡是"不可跨页切断"的块必须带 `resume-item` 基类或 `data-no-break`
4. 打印隔离依赖 `body.builder-printing` + `#root` 直接子节点结构(见 `builder.css`);若未来给 App 外层再包 DOM 容器,打印 CSS 选择器要跟着改
5. `css-modules.d.ts` 是全局声明,删了 Builder 全目录类型报错
6. adapter 在存储边界做字段级默认合并(`normalizeResumeData`),新增消费字段先在这里补默认值,**不要**在模板组件里加 `?? 兜底`(CLAUDE.md §2.1)
7. RM 源在 `reference/projects/`(Git 忽略),文档引用它时写相对路径即可,别把 RM 源码拷进我方仓库提交
