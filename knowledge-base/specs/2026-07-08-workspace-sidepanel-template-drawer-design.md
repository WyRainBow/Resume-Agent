# Workspace SidePanel 重构:模板管理抽屉化 —— 设计

- **日期**:2026-07-08
- **分支**:`feature/template-market`
- **发起**:用户反馈 `/workspace` 编辑区左侧栏(`SidePanel`)太挤、模板选择区不够突出

## 一、现状问题(已核实)

`frontend/src/pages/Workspace/v2/SidePanel/index.tsx` 是三栏布局(`EditPreviewLayout.tsx`)的第一列,**写死 300px 宽**(`sidePanelWidth` 是 `useState` 但没有 setter,实际是常量),纵向堆叠三大块:

1. 「布局设置」—— 模块导航列表(基本信息/教育经历/.../自我评价/竞赛科研 + 添加自定义模块),需要跟中间 `EditPanel` 联动切换,是高频操作
2. 「模板」—— 两组模板缩略图选择(原生 LaTeX 1 个 + Builder 移植的 HTML 模板若干),夹在中间不起眼
3. 「排版设置」—— 条目非常多且**因选择的模板类型而不同**:选 HTML 模板渲染 Builder 的 `FormattingControls`(页面尺寸/边距/间距档位/字号/字体等);选 LaTeX 渲染原生排版卡(字体 pt/英寸边距/行间距/两种经历间距/页面内边距/头部空白三连数字输入)

**根因拆解**:
- 「挤」= 300px 宽度对这些控件(尤其排版设置的下拉/数字输入)来说太窄
- 「长」= 三大块纵向堆叠,排版设置本身条目就多
- 模板/排版不是平级模块,是**因果关系**——排版字段集由模板选择决定,拆开会割裂这个因果可见性

## 二、决策

### 2.1 信息架构:模块导航与模板/排版分层

`SidePanel` 只保留「布局设置」(模块导航),变成纯粹的"我在编辑哪个模块"入口。「模板」+「排版设置」合并成一个新的**「模板与排版」抽屉**,从 `SidePanel` 里移出。

### 2.2 入口位置:SidePanel 顶部横条(非顶部工具栏)

评审时讨论过两个方案:
- **方案 A**:入口挪到页面顶部 `Header` 工具栏,抽屉从右侧滑出,预览列完全不受影响,一次性解决"挤/长/层级错位"三个问题,但改动面更大(要动 `Header` + `index.tsx`)
- **方案 B**:入口留在 `SidePanel` 顶部(工具栏下方、模块列表上方,目前是空白区域),抽屉从这里展开但**加宽到不受 300px 限制**

**选方案 B**。原因:这个入口位置已经用截图跟用户逐条确认过,是明确拍板的细节;方案 A 虽然更彻底,但会推翻这个已确认的位置决策,且改动面不必要地扩大到 `Header`。方案 B 只要把"抽屉宽度"这一个关键盲点修正(不能还是 300px),就能达到同等效果。

### 2.3 抽屉尺寸与遮盖范围

- 收起态:一条常驻横条,显示「模板与排版」+ 当前模板名称/缩略图作为状态指示(不能做成纯图标,否则发现性太差——见评审风险 3),点击展开
- 展开态:浮层宽度 **480-520px**,从 `SidePanel` 顶部展开,**盖住第一列模块导航 + 压住第二列 `EditPanel` 左侧一部分**,第三列 `PreviewPanel` 完全不受影响、全程可见(用户调排版时的实时反馈就靠它)
- 关闭方式:点击横条收起 / 点击浮层外部区域 / `Esc` 键,三种都要支持
- z-index 需要和 `EditPreviewLayout.tsx:338` 现有的拖拽遮罩层(`z-[9999]`,用于列宽拖拽时的全屏事件拦截)协调,抽屉浮层用低于它的层级,避免拖拽列宽时和抽屉遮罩打架

### 2.4 抽屉内部布局:不拆 Tab,同屏纵向

**不做 Tab 切换**。理由(因果可见性优先于紧凑):模板决定排版引擎、引擎决定排版字段集,同屏纵向"模板选择在上、排版设置在下"能让用户亲眼看到"我从 LaTeX 换成 HTML → 下面整组字段变了";拆 Tab 会把这个因果关系藏起来,切换 Tab 后字段"悄悄"变了,反而更困惑。

- 上半部分:模板选择,两组常驻展开(原生 LaTeX / HTML 模板),复用现有 `TemplateOptionButton` + `TemplateThumbnail`
- 下半部分:排版设置,根据当前 `templateType` 显示对应的字段集(HTML → `FormattingControls`;LaTeX → 原生排版卡),**复用现有的 `SettingCard collapsible` 折叠分组**收敛长度(比如"头部空白"这类次要项默认折叠)
- 抽屉整体可纵向滚动(排版字段多是既定事实,加宽只解决横向局促,纵向仍需要滚动,这是可接受的)

## 三、涉及文件与改动范围

| 文件 | 改动 |
|---|---|
| `frontend/src/pages/Workspace/v2/SidePanel/index.tsx` | 主战场:拆出「模板与排版」为独立的抽屉子组件(可以是本文件内的新组件,或拆分成 `TemplateDrawer.tsx` 视代码量决定),`SidePanel` 主体只保留布局设置区块 + 顶部新增横条入口 |
| `frontend/src/pages/Workspace/v2/EditPreviewLayout.tsx` | 可能需要为抽屉浮层提供定位锚点/z-index 协调(具体要不要动,由 `writing-plans` 阶段结合实现细节判断) |

**不动**:`Header`(工具栏)、`PreviewPanel`、`EditPanel/` 下任何具体表单文件、`Builder/` 目录下任何现有文件(`FormattingControls`/`TemplateSelector`/`TemplateThumbnail` 只读复用,不改)。

## 四、决策留痕

| 决策 | 结论 | 理由 |
|---|---|---|
| 模板/排版要不要分开两个入口 | 不分开,合并一个抽屉 | 排版字段是模板选出来的下游,分开会割裂因果可见性 |
| 抽屉内部要不要 Tab | 不要 | 同上,Tab 会藏起因果关系;长度问题靠折叠分组+加宽解决,不靠 Tab |
| 入口放哪 | SidePanel 顶部横条(不挪到 Header) | 用户已用截图明确确认过这个位置;避免不必要扩大改动面 |
| 抽屉宽度 | 480-520px,不是 300px | 300px 是"挤"的根因,抽屉如果照搬这个宽度等于没解决问题 |
| 默认展开还是收起 | 默认收起 | 模板/排版是低频"进入调完退出"的操作,不该常驻占用屏幕空间;但横条要带状态指示,不能做纯图标,否则发现性太差 |
