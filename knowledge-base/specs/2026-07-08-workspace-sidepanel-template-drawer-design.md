# Workspace SidePanel 重构:模板与排版折叠区块化 —— 设计

- **日期**:2026-07-08
- **分支**:`feature/template-market`
- **发起**:用户反馈 `/workspace` 编辑区左侧栏(`SidePanel`)太挤、模板选择区不够突出

## 一、现状问题(已核实)

`frontend/src/pages/Workspace/v2/SidePanel/index.tsx` 是三栏布局(`EditPreviewLayout.tsx`)的第一列,**写死 300px 宽**(`sidePanelWidth` 是 `useState` 但没有 setter,实际是常量),纵向堆叠三大块:

1. 「布局设置」—— 模块导航列表(基本信息/教育经历/.../自我评价/竞赛科研 + 添加自定义模块),需要跟中间 `EditPanel` 联动切换,是高频操作
2. 「模板」—— 两组模板缩略图选择(原生 LaTeX 1 个 + Builder 移植的 HTML 模板若干),夹在中间不起眼
3. 「排版设置」—— 条目非常多且**因选择的模板类型而不同**:选 HTML 模板渲染 Builder 的 `FormattingControls`(页面尺寸/边距/间距档位/字号/字体等);选 LaTeX 渲染原生排版卡(字体 pt/英寸边距/行间距/两种经历间距/页面内边距/头部空白三连数字输入)

**根因拆解**:
- 「挤」= 300px 宽度对这些控件(尤其排版设置的下拉/数字输入)来说太窄
- 「长」= 三大块纵向堆叠,排版设置本身条目就多
- 模板/排版不是平级模块,是**因果关系**——排版字段集由模板选择决定,拆开会割裂这个因果可见性

## 二、决策

### 2.1 信息架构:模块导航与模板/排版分层

`SidePanel` 主体只保留「布局设置」(模块导航),是纯粹的"我在编辑哪个模块"入口。「模板」+「排版设置」合并成一个新的**「模板与排版」折叠区块**,从原来夹在中间的位置移到最上方(「基本信息」模块条目之上)。

### 2.2 位置:SidePanel 顶部(非顶部工具栏)

评审时讨论过把入口挪到页面顶部 `Header` 工具栏、用右侧抽屉展开(改动面更大,要动 `Header`)。**没有采用**:这个"模板与排版"区块的位置已经用截图跟用户逐条确认过,是明确拍板的细节,不做跨组件搬迁,保持改动面集中在 `SidePanel`。

### 2.3 交互形态:默认展开的可折叠区块(非模态浮层)

不是"点击才弹出的模态浮层",是**常驻在 `SidePanel` 列表流里、默认展开的手风琴式区块**,位置在「基本信息」模块条目最上方(布局设置区块之前)。

- **默认展开**:用户一进编辑器就能看到「模板与排版」区块,直接选模板、调样式,不需要先发现"有个东西可以点开"
- **可折叠**:用户点击区块标题栏可以收起,腾出纵向空间去改「基本信息」「教育经历」这些内容模块;需要时随时点开,不是一次性设置完就再也看不见
- **展开时"撑宽",收起时缩回列宽**:展开态视觉上从 `SidePanel` 原位置向右延伸,宽度达到 **480-520px**(压住第二列 `EditPanel` 左侧一部分),解决排版设置控件在 300px 内局促的问题;收起态缩回 `SidePanel` 标准 300px 列宽内,只显示一条状态横条(当前模板名称/缩略图),不占纵向空间
- 第三列 `PreviewPanel` 全程不受影响、全程可见——这是本设计能成立的关键前提:用户调整排版/切换模板时,实时反馈始终看得到
- 折叠/展开由**点击区块标题栏**触发(不是模态框,不需要"点击外部关闭"或 `Esc` 关闭这类模态交互;它是列表流的一部分,常驻存在,只是宽高会变)
- z-index 需要和 `EditPreviewLayout.tsx:338` 现有的拖拽遮罩层(`z-[9999]`,用于列宽拖拽时的全屏事件拦截)协调,展开态覆盖 `EditPanel` 的部分用低于它的层级,避免拖拽列宽时打架

### 2.4 区块内部布局:不拆 Tab,同屏纵向

**不做 Tab 切换**。理由(因果可见性优先于紧凑):模板决定排版引擎、引擎决定排版字段集,同屏纵向"模板选择在上、排版设置在下"能让用户亲眼看到"我从 LaTeX 换成 HTML → 下面整组字段变了";拆 Tab 会把这个因果关系藏起来,切换 Tab 后字段"悄悄"变了,反而更困惑。

- 上半部分:模板选择,两组常驻展开(原生 LaTeX / HTML 模板),复用现有 `TemplateOptionButton` + `TemplateThumbnail`
- 下半部分:排版设置,根据当前 `templateType` 显示对应的字段集(HTML → `FormattingControls`;LaTeX → 原生排版卡),**复用现有的 `SettingCard collapsible` 折叠分组**收敛长度(比如"头部空白"这类次要项默认折叠)
- 区块整体可纵向滚动(排版字段多是既定事实,加宽只解决横向局促,纵向仍需要滚动,这是可接受的)

## 三、涉及文件与改动范围

| 文件 | 改动 |
|---|---|
| `frontend/src/pages/Workspace/v2/SidePanel/index.tsx` | 主战场:拆出「模板与排版」为独立子组件(可以是本文件内的新组件,或拆分成 `TemplateAndLayoutSection.tsx` 视代码量决定),挂载到「布局设置」区块之前;`SidePanel` 主体只保留布局设置区块 |
| `frontend/src/pages/Workspace/v2/EditPreviewLayout.tsx` | 可能需要为展开态的覆盖范围提供定位锚点/z-index 协调(具体要不要动,由 `writing-plans` 阶段结合实现细节判断) |

**不动**:`Header`(工具栏)、`PreviewPanel`、`EditPanel/` 下任何具体表单文件、`Builder/` 目录下任何现有文件(`FormattingControls`/`TemplateSelector`/`TemplateThumbnail` 只读复用,不改)。

## 四、决策留痕

| 决策 | 结论 | 理由 |
|---|---|---|
| 模板/排版要不要分开两个入口 | 不分开,合并一个区块 | 排版字段是模板选出来的下游,分开会割裂因果可见性 |
| 区块内部要不要 Tab | 不要 | 同上,Tab 会藏起因果关系;长度问题靠折叠分组+加宽解决,不靠 Tab |
| 位置放哪 | SidePanel 顶部,基本信息模块之上(不挪到 Header) | 用户已用截图明确确认过这个位置;避免不必要扩大改动面 |
| 展开态宽度 | 480-520px,不是 300px | 300px 是"挤"的根因,如果照搬这个宽度等于没解决问题 |
| 交互形态 | 常驻列表流内的可折叠区块,不是模态浮层 | 用户明确要求"默认展开、可以随时收起去改内容、随时再点开看效果",这是手风琴式而非弹层式交互 |
| 默认展开还是收起 | 默认展开 | 提升发现性——用户一进编辑器就知道可以换模板/调排版,不需要先发现"这里能点开" |
