**这些绝不能删**——删了会破坏 Workspace 的模板渲染。是否从 UI 上隐藏双栏选项是另一个产品决策，但不影响本次页面删除。

## 五、删除方案（建议）

### Phase 1：删页面 + 路由（安全，不影响 Workspace）

1. **App.tsx**：删 3 个 lazy import + 4 条路由 + 注释
2. **删 3 个页面文件**：`Builder/index.tsx`、`DashboardPage.tsx`、`SettingsPage.tsx`
3. **删 forms/ 全部 11 文件**：只被 index.tsx → ResumeForm 用
4. **删 api.ts**：无引用的死代码
5. **删 builder.css**：只被 index.tsx 用
6. **删 4 个独用组件**：PageContainer / RetroTabs / SwissButton / usePagination

### Phase 2：验证

- `npm run build` 通过
- Workspace `/workspace/html` 正常渲染（模板/预览/格式控制不受影响）
- `/builder` 路由 404（预期）

### 不动的部分

- `templates/` 全部保留（Workspace 用）
- `components/` 的 FormattingControls / PaginatedPreview / TemplateSelector 保留（Workspace 用）
- `adapter.ts` / `settings.ts` / `pageDimensions.ts` / `types.ts` / `css-modules.d.ts` 保留

## 六、风险评估

| 风险 | 等级 | 说明 |
|---|---|---|
| Workspace 模板渲染受影响 | 低 | 删的都是页面层，模板/组件/适配器全保留 |
| 外部链接到 /builder 404 | 低 | `/builder` 是新功能，外部几乎无引用；可加重定向到 `/workspace/html` |
| forms/ 误删 | 低 | 已确认只被 index.tsx → ResumeForm 引用，Workspace 用的是自己的 EditPanel |
| 双栏模板遗留 | 不影响本次 | 属于另一个产品决策 |
| forms/shared.tsx 反向依赖 | 不影响 | `shared.tsx:7` import 了 Workspace 的 RichEditor（Builder→Workspace 反向依赖），但 forms 整体要删，不影响删除决策。说明 Builder 与 Workspace 有双向耦合，未来若反向搬迁需注意 |

**结论**：删除安全，Workspace 能力零影响。核心依据是 Workspace 引用的都是 `templates/` + `components/`(3个) + `adapter.ts` + `settings.ts` + `pageDimensions.ts`，这些全不在删除清单里。
