# upcv 简历诊断功能体验分析

> 分析日期：2026-03-27
> 来源：https://upcv.tech

## 1. 交互流程

### 1.1 入口与触发
- 首页有明确的"简历诊断"快捷按钮
- 用户也可以直接输入"帮我看看简历有什么问题"触发诊断

### 1.2 两阶段确认流程
1. **第一阶段**：获取简历后，先询问目标岗位
   - 展示三个选项按钮：
     - "告诉我目标岗位名称 →"
     - "发送目标职位的 JD 给我 →"
     - "先做通用简历诊断 →"
2. **第二阶段**：用户选择后，开始详细诊断

## 2. 思考过程 (Thought) 展示

### 2.1 Thought 组件
- 折叠式 "Thought" 按钮，点击展开
- 显示 "Thought for Xs" 时间戳（我们已去掉）
- **关键特点**：思考过程是**流式输出**的，边思考边展示

### 2.2 思考内容结构（非常详细）
```
好的，用户选择了先做通用简历诊断，那我就基于已获取的简历内容，按照诊断检查清单逐项扫描，然后输出三维评分和问题清单。

先快速过一遍简历内容：

基本信息：
姓名：王嘉毅 ✓
邮箱：wangjiayi.dev@163.com ✓（格式正常）
电话：188-1110-0166 ✓（格式正常）
简历标题：王嘉毅_后端开发_应届 ✓（包含姓名）
照片：url 为空，但 effects 配置了，这个不算问题

教育经历：
1 段教育经历 ✓
学校：中央民族大学（985）✓
专业：计算机科学与技术 ✓
学位：本科 ✓
时间：2022.09 - 2026.06 ✓（预计毕业时间，不是"至今"）

工作经历：
3 段工作经历（字节、快手、美团）✓ 不超过 4
...
```

**特点**：
- 思考过程非常**结构化**，按模块逐项检查
- 每项检查都有 ✓ 或具体问题标注
- 像 HR 一样逐行扫描简历

## 4. 诊断报告结构

### 4.1 初筛通过概率
```
以当前简历投递互联网后端开发岗位，初筛通过概率约 85%（竞争力很强，但有小瑕疵需要优化）
```

### 4.2 评分卡（三维）
```
内容质量：82/100 — 整体专业规范，量化数据充分，但存在内容重复和空模块问题
竞争力：92/100 — 985 本科 + 三段大厂实习经历...
岗位匹配度：未评估 — 你还没告诉我目标职位...
```

### 4.3 问题清单（分级）

**必须修改（影响专业度）**
- 同一项目重复出现 3 次（项目经历、社团、其他经历）
- 多个模块为空但可见

**建议修改（提升专业度）**
- 简历标题可以更专业
- 教育经历缺少 GPA/排名
- 头像为空

**可选优化（锦上添花）**
- 个人总结可以更精炼
- 技能等级标识

### 4.4 Top 3 优化建议
```
1. 删除重复内容
2. 隐藏空模块
3. 补充教育成绩
```

### 4.5 下一步 CTA
```
A. 我帮你直接修改这些重复内容和空模块问题
B. 先看看有哪些匹配的后端开发职位，然后针对性优化
C. 针对某个具体公司的 JD 再做一版定向简历
```

## 5. 与我们系统的对比

| 维度 | upcv | 我们的系统 |
|------|------|------------|
| 思考过程 | 流式输出，边想边展示，按模块逐项扫描 | 静态一次性输出，4个步骤 |
| 询问策略 | 先问目标岗位，再做诊断 | 直接开始诊断 |
| 评分维度 | 内容质量、竞争力、匹配度 | 类似但描述略有不同 |
| 问题分级 | 必须/建议/可选，非常清晰 | 必须修改/建议修改/可选优化 |
| CTA 按钮 | A/B/C 三个明确选项 | %%SUGGESTIONS%% 格式 |

## 6. 值得借鉴的设计

1. **流式思考过程**：思考是边想边输出的，不是等全部想完再输出
2. **结构化检查清单**：像 HR 一样逐项扫描简历
3. **两阶段确认**：先问目标岗位，提供更好诊断
4. **明确的 CTA**：A/B/C 选项比纯文本建议更直观
5. **问题位置标注**：指出具体问题在简历的哪个位置

## 7. 可能的 prompt 风格

从思考过程看，upcv 的 system prompt 可能包含：
- "按照诊断检查清单逐项扫描"
- "先快速过一遍简历内容"
- "输出三维评分和问题清单"
- "指出具体位置"
