# AI简历评分功能设计

## 概述

输入：简历 + JD（用户粘贴的职位描述）
输出：3维度评分 + 各维度匹配/不匹配原因

## 评分维度

| 维度 | 计算方式 | 分数范围 |
|------|----------|----------|
| 技能与经验匹配 | LLM分析 JD要求 vs 简历技能+经历 | 0-100 |
| 教育背景匹配 | LLM分析 JD学历要求 vs 简历教育背景 | 0-100 |
| 项目与整体匹配 | Embedding向量相似度 + LLM补充分析 | 0-100 |

额外输出：总体匹配度（3维度加权平均）、各维度匹配/不匹配原因

## 用户交互流程

1. 用户在简历详情页粘贴 JD 文本
2. 上传简历后自动触发评分（无需手动点击）
3. 前端展示评分卡片：3个维度分数 + 总体匹配度 + 原因分析

## 技术方案

### 后端

**新增接口：**
- `POST /api/resume/score` — 评分主接口
  - 请求体：`{ resume_id: str, jd_text: str }`
  - 响应：`{ overall_score: float, dimensions: [{ name, score, reasons }] }`

**复用现有能力：**
- `embedding_service.py` — 生成向量相似度
- `call_llm()` — LLM 多维度分析
- `Resume` / `ResumeEmbedding` 数据模型

**新增数据模型：**
- `ScoreResult` — 存储评分结果（resume_id, jd_text, scores, created_at）

### 前端

- 在简历详情页嵌入评分卡片组件
- 上传简历后自动调用评分接口
- 展示3维度分数条 + 总体匹配度 + 原因列表

## 评分算法

### 技能与经验匹配
1. 提取 JD 中的技能关键词和工作年限要求
2. 提取简历中的技能列表和工作经历
3. LLM 对比分析，输出分数(0-100)和匹配/不匹配原因

### 教育背景匹配
1. 提取 JD 中的学历要求（本科/硕士/博士等）
2. 提取简历中的教育信息（学校、学历、专业）
3. LLM 对比分析，输出分数(0-100)和原因

### 项目与整体匹配
1. 计算简历全文与 JD 的 Embedding 向量余弦相似度 → 基础分
2. LLM 分析项目经历与 JD 相关性，给出调整分
3. 最终分数 = 基础分 * 0.4 + LLM调整分 * 0.6

### 总体匹配度
```
总体匹配度 = 技能与经验匹配 * 0.4 + 教育背景匹配 * 0.2 + 项目与整体匹配 * 0.4
```

## 依赖

- 复用 `embedding_service.py` 的 `get_embedding()` 方法
- 复用 `llm.py` 的 `call_llm()` 方法
- 复用 `resume_router.py` 的解析逻辑

## 边界情况

- JD 文本为空：返回错误提示
- 简历解析失败：返回错误提示，不影响上传
- LLM 调用超时：降级为纯 Embedding 分数
- 重复评分：覆盖历史评分结果
