## 一、方法论（做任何功能前对照）

### 刘小排核心思想（浓缩版）

| 原则 | 一句话 |
|---|---|
| 真需求 = 愿意付出代价 | 越靠近钱、时间、风险、职业机会，付费意愿越强 |
| 订阅是问题的时间结构 | 一年一次的问题撑不起订阅——简历天然低频，要么单次收费，要么把低频做成高频 |
| 核心动作明确 | 用户一句话能理解产品；一屏回答一个问题、一个主 CTA |
| 买时间 | 首屏只争取「继续看下去」；每一步只争取下一步 |
| 结果直接可见 | 做完立刻看到效果（右侧预览 / 对比卡 / before-after） |
| 价值先行、付费后置 | 付费引导放在「已感受到价值 + 刚好需要更高权益」的时刻，不突然弹窗 |
| 黄金规则 | 用户明确说要做的事直接动手，不反复确认；范围模糊才问一次 |
| 打磨优先级 | ①商业闭环（首屏→登录→额度→支付→权益）②增长（SEO/案例）③高级功能（团队/多租户）——顺序不能反 |
| 静默失败比不提供更糟 | 每个失败都要让用户知道 + 能重试 |
| 真正上线 ≠ 部署成功 | 真实用户能从看到页面走到拿到权益，链路断一处都不算上线 |

### 我的补充（本项目实践中沉淀）

1. **最短价值闭环**：导入简历 → 一键整份优化 → 全部应用 → 下载 PDF。所有入口都往这条链路上导。
2. **透明化**：范围模糊的指令（「优化简历」）先出计划让用户选；明确指令（「优化整份」）直接执行。过程渐进可见（对比卡逐张出现、完成后列出改了哪几段）。
3. **可撤销 > 二次确认**：能撤销的操作不拦人；不可逆的操作才上应用内确认框。
4. **生产构建实测**（2026-07-04 TDZ 事故教训）：`useCallback` deps 引用后置声明在 dev 能跑、生产压缩后整页崩溃。**改完对话页必须用生产构建（`vite preview`）或线上冒烟一次核心路由。**
5. **可靠验证纪律**：push 后 `git ls-remote` 核对；线上版本用**字面**查找（grep 的 `.` 是正则，`2.8.0` 会误匹配）；部署失败先看服务器 `auto-deploy.log`。

## 三、未做的建议（按优先级）

### P0 · 体验硬伤

1. **自动保存常驻状态指示器**：失败已有 toast，但缺「保存中… / 已保存」常驻小字（`useAutoSaveResume` 的 `saveStatus` 已算好没人渲染）；且主工作台 `v2/index.tsx` 没有服务端自动保存、latex/html 有——**两套保存模型该统一**。

### P1 · 核心动作再进一步

2. **整份优化的 review 负担**：8 张对比卡滚动过长。方案：卡片默认折叠成一行摘要（段名 + 一句变化），点开看 diff；「全部应用」置顶常驻。
3. **优化覆盖面补全**：教育经历、荣誉奖项等 section 目前明确回应「不支持」，可按技能/自我评价的单字段模式补上。
4. **PDF 渲染报错横幅易逝**：已可重试，但报错横幅仍会被下一次渲染顶掉；改常驻错误态直到成功。

### P1 · 产品价值与增长基础

5. **低频 → 高频**（建议 0，战略级）：一份主简历 → 按不同 JD 生成多版本并管理；投递记录；版本历史/对比。这是把「一年用一次」变成「求职季每周用」的关键。
6. **Landing 首屏 before/after 视觉**：现在还是产品截图，换成「一句话经历 → 生成简历」对比（细节模糊化，参照 Simplify）。

### P2 · 增长

7. SEO（标题/描述/OG 分享图）、案例页、博客。

### P2 · 工程质量（防回归）

8. **auto-deploy.sh 加 flock 防并发**（2026-07-04 实测两个实例同时构建）+ 把脚本纳入仓库管理（现在是服务器上的孤本）。
9. **生产构建冒烟**：CI 或本地脚本跑 `vite build && vite preview`，无头浏览器开 `/`、`/agent/new`、`/workspace` 三条路由查 console error（防 TDZ 类只在压缩产物暴露的崩溃）。
10. code-stats workflow 并发撞锁修复（已改 `concurrency` + push 重试，随本文档一起提交）。

### 暂缓 · 明确不做（不要再出现在建议清单里）

- **商业闭环 / 支付链路**（付费引导时机、额度边界提示、导出/高清/去水印升级引导、价值峰值付费点等）：用户于 2026-07-04 明确决定**暂时不做**，后续打磨建议中不要再列入此类条目，直到用户主动重启该方向。
- **移动端适配**（对话页响应式、预览面板手机布局等）：用户于 2026-07-04 明确决定**暂时不做**，同上，不再列入建议。原问题记录备查：`CocoChat.tsx` 全文无响应式断点；PDF 预览面板 `w-[45%] min-w-[420px]`（`AgentPdfPreviewPanel.tsx:85`）在手机上吃满视口。
