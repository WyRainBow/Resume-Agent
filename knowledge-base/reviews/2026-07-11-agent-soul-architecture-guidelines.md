# Agent 架构准则:怎么做出"有灵魂"的 Agent

- 日期:2026-07-11
- 用途:给协作的其它 AI(或未来的自己)参考的准则,基于今天一整天的架构审计+两轮外部范式调研+两次真实代码问题(一次是仓库自身的规则引擎、一次是另一个 AI 新提交的 prompt 改动)总结。**这不是抽象理论,每条都对应过一次真实踩坑。**

---

## 一、核心判据轴(唯一标准)

**判断一段逻辑该写成"规则"还是该交给"LLM 自主判断",标准只有一条:这是在管安全/防呆/成本边界,还是在替 LLM 理解用户意图、决定怎么做。**

- **规则该管**:不可逆操作要人类确认、敏感数据(隐私字段)默认脱敏、成本/延迟边界(比如高频确定场景走 fast path)、参数合法性校验。
- **规则不该管**:用户想要什么、该优化简历的哪个模块、该用什么语气回复、该不该追问用户、这份简历改得好不好。这些是理解意图和专业判断,是 LLM 的活。

一句话:**规则产生服从,价值观产生性格**。守规则的 Agent 会在规则没覆盖的边缘失灵,内化了价值观的 Agent 能自然应对边缘。

---

## 二、两次真实踩坑(同一类问题的两种介质)

### 踩坑 1:Python 代码里的规则引擎(今天删掉的)

`AGENT_LLM_FIRST_ROUTING` 默认已经打开,但代码里还留着一整套规则引擎(`resume_use_cases.py` 1172 行、`experience_entry.py` 里"猜用户指哪段经历"的模糊匹配函数)——这些代码在默认配置下根本不会执行,是死代码,但团队还在维护、还在重构它。删了,连同开关本身,不留退路。

### 踩坑 2:prompt 文本里的规则清单(今天下午撞见的)

另一个 AI 在同一个分支上提交了一版"整份优化增强",在 system prompt 里加了"必须逐模块覆盖,不允许只挑经历跳过技能"这种**强制清单表格**,还有"必须发挥自主性,不允许只挑 2-3 段经历就收尾"这种禁令式措辞。

**这和踩坑 1 是同一类问题,只是介质从 Python 的 if/elif 换成了 prompt 里的 markdown 表格。硬编码没消失,只是换了个地方藏。**"该优化哪些模块"本就是理解意图的活,该让 LLM 读完简历自己判断,不该用一张写死的表格逼它每次都要覆盖全部模块——用户简历某个模块本来就写得好,这种"不允许跳过"会导致 Agent 画蛇添足瞎改。

**结论:检查一段 prompt 或代码是不是"规则堆砌",不能只看它是 Python 还是文本,要看它的语气是"讲清楚期望、让 LLM 自己判断"还是"规定必须按什么顺序做什么、不允许怎样"。**

## 三、价值观式 vs 规则清单式(对比示例)

| 规则清单式(该避免) | 价值观式(该追求) |
|---|---|
| "必须逐模块覆盖,不允许只挑经历跳过技能" | "通读全简历,自己判断哪些模块有提升空间就动哪些;模块本来就写得好就不要硬改" |
| "按方向补充 AI Native 能力"+ 一张必须照抄的固定清单 | "结合当下主流的 AI Native 能力,挑真正契合这份简历的补充进去,不要机械罗列" |
| "不要说'我具备…''本人熟悉…'这种学生表达" | (这条本身就是价值观式,该保留——它讲的是审美判断,不是流程步骤) |

**判断口诀**:写完一段 prompt/代码,问自己——"这是在讲清楚期望让 LLM 自己判断,还是在规定它必须按什么顺序做什么、不允许怎样?" 后者要改。

## 四、隐私/安全类规则的正确姿势

隐私脱敏这种**该**由规则管(安全边界),但今天撞见的实现方式本身也错了两次:

1. **不要重复造判断表**:项目里已经有意图判定机制(`Intent.FULL_OPTIMIZE` 这类),不要在别的地方另起一张关键词表去猜"是不是整份优化场景"——两张表迟早对不上,今天就实测撞见了"用户说'全局优化'触发了意图判定但没触发脱敏"这种 bug。
2. **别只堵一个出口**:如果只在 prompt 注入这一层脱敏,LLM 主动调用读取工具时照样能拿到完整隐私数据。安全边界要在数据出口层统一处理,不能只挡一条路。

---

## 五、垂直 Agent 的定位纪律

窄工具集(简历场景就 7 个工具:读/展示/诊断/编辑/生成/发邮件/追问)本身就是"灵魂"的一部分——工具集边界越清晰,LLM 的自主决策越聚焦、越像专家而不是万金油助手。**不要为了"功能完整"给 Agent 塞更多工具或更多判断分支,窄而深比广而浅更有主见。**

---

## 六、给协作 AI 的一句话总结

**写 Agent 相关代码/prompt 前,先问:这段东西是在告诉 LLM"世界是什么样、你该怎么想",还是在告诉它"你必须按这个顺序做这些事、不许那样"?前者留,后者删或者改写成前者。规则只配管钱、管安全、管不可逆操作;理解用户想要什么,永远是 LLM 的活。**
