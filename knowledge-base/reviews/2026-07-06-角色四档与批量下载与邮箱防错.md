## 1. 角色体系：管理员 / 员工 / 会员 / 普通用户（`2e577dcb`）

**改前问题**：只有 admin/member/user 三档，member（会员）错误地承担了内部运营权限——能进后台看全量用户列表/统计（PII）、改系统提示词；同时会员没有任何真实付费权益（PDF 下载和普通用户一样 10 次）。

**改后权限矩阵**（本地后端实测，四角色 × 关键接口）：

| 接口 | user | member | staff | admin |
|---|---|---|---|---|
| GET /api/admin/users（用户列表） | 403 | **403**（原 200） | **200** | 200 |
| GET/PUT /api/config/prompts | 403 | **403**（原 200） | **200** | 200 |
| PATCH /api/admin/users/:id/role | 403 | 403 | 403 | ✓（仅 admin） |
| logo / 校徽上传 | 403 | **403**（原可传） | 200 | 200 |
| PDF 下载额度 | 10 次 | **不限次**（新权益） | 不限次 | 不限次 |
| 匿名访问后台 | 401 | | | |

**实现**：`require_admin_or_member` → `require_staff`（admin|staff）；`pdf_download_quota.UNLIMITED_ROLES = {admin, staff, member}`；前端 `canUseAdminFeature()` 收窄为 admin|staff；后台角色下拉/统计卡/图例支持四档。生产库改动时 member 0 人，无存量影响。

**会员定位**：纯付费权益位。当前唯一权益 = PDF 下载不限次；后续权益等商业闭环重启再加（商业闭环本身仍暂缓，见路线图「暂缓」节）。

## 2. 「我的简历」一键下载全部 / 批量下载（`8ec96c92`）

- 常态 Header 新增「下载全部」；多选模式选中后出现「下载」（与批量删除同位、同交互）。两者共用 `batchDownload(ids?)`。
- 流程：额度预检（不足先拦）→ 确认框（明示消耗 N 次、剩余 M、HTML 跳过数）→ 逐份 `convertToBackendFormat` + `/api/pdf/render` → jszip 打包 `我的简历-YYYY-MM-DD.zip`（UTF-8 文件名，按 姓名-备注 命名、重名加序号）→ 成功后按份调 `/pdf/downloads/record` 记账 → 结果 toast（成功/跳过/失败数）。
- HTML 模板简历不走 LaTeX 渲染链，批量时跳过并提示；单份失败不中断。
- **实测抓到并修复一个真 bug**：仪表盘直接用存储原始数据，缺 `menuSections` 时 `convertToBackendFormat` 崩（`menuSections.reduce`）；修复为按工作台 `initialResumeData` 默认值合并后再转换（存储读取=系统边界校验）。
- 浏览器实测：确认框额度文案正确 → zip 落盘（2 份真 PDF、中文名 UTF-8 flag 正确）→ 额度 used 0→2。`unzip -l` 中文乱码是其 CP437 显示问题，非数据问题。

## 3. 注册邮箱「重复校验」排查结论 + 拼写防错（`27c44293`）

**结论：查重本来就有，且生效。** BetterAuth `user.email` 有数据库级唯一索引（`user_email_key`），重复注册报「User already exists」且登录页会显示。生产 15 个用户 0 重复。截图里两条 cocoyu 其实是 **`gamil.com`（拼写错误）和 `gmail.com` 两个不同地址**——同一个人第一次把域名拼错了。

**真实缺口**：拼错域名照样注册成功 → 孤儿账号（收不到任何邮件、忘密码无法找回）→ 用户只好换对的邮箱再注册一个，后台看起来像「重复」。

**本次落地（轻量防错）**：登录页（web/BetterAuth 面板）内置高频错拼域名表（gamil/gmial/gmail.con/qq.con/163.con/outlok/hotmial/iclould 等 19 条）：输入时实时出现「邮箱可能拼写有误，改为 @gmail.com」一键修正按钮；提交时软拦截一次（再点一次可坚持使用）。本地 dev 实测：提示出现、一键修正、软拦截文案均正常，未发起真实注册。

**未做（如需再立项）**：邮箱真实性验证（`requireEmailVerification` + 邮件服务商），能根治但需要 SMTP/Resend 基础设施。
