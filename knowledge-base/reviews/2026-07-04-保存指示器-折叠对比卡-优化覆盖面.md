## 1. 自动保存常驻状态指示器（commit `a1d8ebe8`）

**改动**：
- 新增 `frontend/src/pages/Workspace/v2/components/SaveStatusIndicator.tsx`：`pending/saving` → 「保存中…」（转圈）、`saved` → 「已保存」（绿勾）、`error` → 「保存失败」（红，title 带错误信息）、`idle` → 不渲染。
- 共享 `Header.tsx` 增加可选 `saveStatus` / `saveError` props，指示器渲染在保存按钮左侧；`useAutoSaveResume.ts` 导出 `AutoSaveStatus` 类型。
- latex 工作台（`latex/index.tsx`）把已有 hook 的 `saveStatus` 接到 Header（此前算好没人渲染）。
- **统一保存模型**：主工作台 `v2/index.tsx` 接入与 latex/html 相同的 `useAutoSaveResume`；「保存成功快照」effect 改为只依赖保存状态变化（`saveSuccess` / `saveStatus === 'saved'` 的瞬间），避免把保存后的新编辑误标为已保存。
- html 工作台无 Header 顶栏，本次不加视觉指示器（外科式修改，它本就有自动保存）。

**关键事实**：`resumeStorage.saveResume` 按登录态分流（未登录走 localStorage 适配器），主工作台加自动保存不会对匿名用户产生 401 轰炸——方案可行性的前置确认。

**验证**：`npm run build` + `vite preview` 生产构建 + Playwright 实测 `/workspace`：编辑姓名 → 头部出现「已保存」；`/workspace/latex` 正常加载；console 无新增错误（仅后端未启动的既有 API 失败）。

## 3. 优化覆盖面补全：教育经历 / 荣誉奖项（commit `bafb5005`）

**改动**：
- `backend/agent/utils/experience_entry.py`：`SectionKind` 扩展 `education` / `awards`；`_iter_optimize_sections` 接入 `education` / `awards` 数组（value_field 均为 `description`）；`_entry_has_optimizable_content` 约束教育/奖项**仅有描述正文的条目可优化**（实习/项目/开源保持原行为）；关键词映射（教育/学历/荣誉/奖项/获奖）、优化计划清单、澄清列表、精确匹配全部同步。
- `backend/agent/agent/manus.py`：section 收窄分支纳入 education/awards（空描述时回「先补一段描述」）；`_llm_optimize_section_patch` 对教育/奖项追加提示——不套经历四要素，改写为亮点描述、不编造；section 中文名改 dict 映射。

**顺带修复：section 点名被上下文劫持（实测中发现的真实 bug）**
- 现象：发「优化荣誉奖项」产出的却是上一轮讨论的教育经历优化卡。
- 根因：OPTIMIZE_SECTION 分支先跑 `resolve_optimize_target`（输入精确匹配失败后回退 **recent_assistant 上下文推断**），命中即短路、section 类型检测被跳过。「优化荣誉奖项」不在泛化词表 → 精确匹配 None → 上下文匹配到上轮消息里的「某大学」→ 劫持。旧 section 同样中招（讨论完实习后说「优化项目经历」）。
- 修复：先 `detect_optimize_section_kind`；**用户点名 section 类型时只做输入内精确匹配（`resolve_optimize_target_from_input`）、禁用上下文推断**。`is_generic_optimize_section_query` 词表补全全部 section 类型词，防 section 词误匹配条目名。

## 4. 验证环境与踩坑（可复用）

- **本地全链路**：后端 `.venv/bin/python -m uvicorn backend.main:app --port 9000`（⚠️ 本地 `.env` 直连**生产 PostgreSQL**，任何写操作要用测试账号并事后清理）；前端 `VITE_API_VIA_AUTH_WEB=false VITE_AUTH_WEB_URL= npm run build && npx vite preview --port 4173`（shell 变量覆盖 `.env.local`，不落盘）——否则构建把「API 经 localhost:3000 Next 反代」烘进产物，preview 下全部请求失败。
- **测试登录**：`POST /api/auth/register` 造临时用户拿 JWT → Playwright 往 `localStorage` 写 `auth_token` + `auth_user`（legacy JWT 模式）。注意先导航到同源页面再 evaluate，否则被登录页重定向到未运行的 localhost:3000 报 SecurityError。
- **⚠️ `&&` 链坑（本次实际踩到）**：`python -c "..." && pkill ... && 重启` 里 python 先失败导致 pkill/重启没执行，后端跑的还是旧代码——表现为「修复无效」。**重启后必须用 `ps -o lstart= -p <pid>` 核对进程启动时间**。
- **beforeunload 弹窗会卡死 Playwright 导航**：工作台有未保存提醒，关页/跳转前处理 dialog。
- **清理入账**：测试用户（id=19 uitest_local）、测试简历、55 条 api_request_logs 已从生产库删净（user_id 精确限定）；会话历史因 append 409 本就未入库；`dist` 已用正常 env 重新构建；临时文件已删。
